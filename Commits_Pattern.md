# Padrãonização de Commits

Padrão de nomenclatura e boas práticas para mensagens de commit no projeto.

---

## Prefixos e Utilização

* `git commit -m "test: ..."` — Adiciona ou modifica testes na aplicação
* `git commit -m "feat: ..."` — Adiciona uma nova funcionalidade ao projeto
* `git commit -m "refactor: ..."` — Refatora o código sem alterar a sua funcionalidade
* `git commit -m "style: ..."` — Altera a formatação ou estilo do código sem mudar a lógica
* `git commit -m "fix: ..."` — Corrige um bug ou erro no código
* `git commit -m "chore: ..."` — Realiza tarefas de manutenção ou alterações em arquivos de configuração
* `git commit -m "docs: ..."` — Adiciona ou altera documentações do projeto
* `git commit -m "perf: ..."` — Realiza alterações focadas em melhoria de desempenho
* `git commit -m "build: ..."` — Altera dependências ou configurações de build do projeto
* `git commit -m "revert: ..."` — Reverte as alterações de um commit anterior
